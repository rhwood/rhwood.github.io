Google Applications

Jan 6th, 2008

Google applications on Mac OS X are installed by default either in the /Applications folder or a subfolder of /Applications (see the current state to the right).

I have every Google application installed:

Google Desktop

Its actually two applications, Google Desktop (a search tool integrating searches on the computer and the internet) and Google Gadgets (allowing widgets that work on the google.com pages to be displayed in the Dashboard) installed in the folder /Applications/Google Desktop.

Google Earth

Its installed in /Applications. Unlike most of the Google applications, it is a complex application in its own right.

Google Notifier

This application places notifiers for GMail and the Google Calendar in the menu bar, and once running is accessed exclusively through the menu bar. Once a user decides to use it, it can be set to automatically start when the user logs in through its preferences window.

Google Sketchup

This is a modeling application. For some reason it is installed in a versioned folder that contains nothing but the application.

Google Updater

The tool that manages all of the Google applications. This tool installs, updates, and removes the other Google applications.

Picasa Web Albums Uploader

An application for importing photos into the [Picasa web albums](http://picasaweb.google.com/). I use the [iPhoto](http://www.apple.com/iphoto) plugin that it installs. (The application is not shown in the listings.)

I want the Google Updater to install all of the applications that run as services on the computer, or are utilities for maintaining the computer, in the folder /Applications/Google Applications (or some similar name).

I don’t know if this just some irritation of mine that I have too many applications on my computer or not, but I just don’t like the current state of affairs.
